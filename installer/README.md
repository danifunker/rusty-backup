# Windows installer (Inno Setup)

`rusty-backup.iss` builds the per-user Windows installer (shipped 2026-06-02
alongside the in-app `self_replace`-based self-updater).

## Why per-user

The installer targets `%LocalAppData%\Programs\Rusty Backup`
(`PrivilegesRequired=lowest`) so no elevation is needed and the in-app
self-updater (`self_replace`) can overwrite the running exe later. A
`Program Files` install would require admin for every update.

## Layout it produces

- `{app}\rusty-backup.exe` — the GUI (kept at the install root; this is what
  the self-updater's `current_exe()` resolves to).
- `{app}\bin\rb-cli.exe` — the CLI, in a subdir so only it lands on PATH.
- `{app}\icon.ico`.

## Tasks (user-facing checkboxes)

- **Add rb-cli to PATH** — default **on**. Adds `{app}\bin` to the per-user
  `HKCU\Environment\Path` (`ChangesEnvironment=yes` broadcasts the change).
  Removed on uninstall.
- **Associate disk image files** — default **on**. Runs
  `rusty-backup.exe --register-file-associations`, which registers the
  extensions from `model::file_types` (single source of truth) under
  `HKCU\Software\Classes` as an `OpenWithProgids` handler (does not seize the
  default). Uninstall runs `--unregister-file-associations`.
- **Desktop shortcut** — default off.

Add/Remove Programs entry + uninstaller are generated by Inno automatically.
After a self-update bypasses the installer, the app refreshes its own ARP
`DisplayVersion` (Phase 5) so the entry stays accurate.

## Building

```
iscc /DMyAppVersion=<version> /DSourceDir=<dir-with-exes> /DAssetsDir=<dir-with-icon> installer\rusty-backup.iss
```

Defaults (`SourceDir=..\target\release`, `AssetsDir=..\assets\icons`) let you
run `iscc installer\rusty-backup.iss` after a local `cargo build --release`.
CI compiles this per arch and attaches `Setup.exe` to the GitHub release.

## AppId

The `AppId` GUID in the script is stable and **must not change** across
releases, or upgrades will install side-by-side instead of replacing in place.
